# Vision

Present a clear, evidence-led professional portfolio as a fast, accessible,
repository-base-path-safe static site that can be published on GitHub Pages without a
production server.

The project favors semantic HTML, restrained visual design, factual provenance, useful
navigation, responsive behavior, and durable plain files over framework or backend
complexity.

## Goals

- Present verified professional experience, projects, and evidence clearly.
- Load quickly and remain usable at the GitHub Pages repository base path.
- Provide keyboard-accessible, responsive, printable, reduced-motion-friendly pages.
- Keep production architecture limited to static HTML, CSS, JavaScript, and assets.

## Non-goals

- Recreating or modifying the `evidence-trail-builder` inspiration repository.
- Adding a server, client router, framework runtime, database, authentication, or CMS.
- Publishing invented identity, employment, education, credential, contact, or evidence claims.
- Treating a local preview server as part of production deployment.
