# Contracts — Static Portfolio Artifacts

The authoritative contracts are the declared page inventory, repository-base-path link rules, forbidden-placeholder policy, dependency boundary, and test assertions maintained by the testing standard. Every validation result and publication artifact is validated against those contracts before acceptance. Missing pages, broken links or assets, placeholder content, server dependencies, or unexpected framework/database files reject the artifact. Durable content decisions route to [content provenance](../content-provenance.md); documentation ownership routes to [documentation](../documentation.md).
