# Resume Portfolio Content Provenance

This document governs the factual content that may appear in the published resume
and portfolio. It is maintained alongside the site and reviewed before publication.

## Source policy

The inspiration repository supplies visual and information-architecture motifs only.
It is read-only and is not a source of personal identity, employment, education,
credential, contact, date, or evidence claims. Personal facts must come from
operator-supplied source material or a verified first-party record.

## Adapted versus new content

Adapted content may include neutral layout ideas, evidence-oriented labels, timeline
patterns, and transparency language. New presentation copy must be authored for this
portfolio and must not imply an unverified accomplishment. Every external link must
be checked before the build is allowed to publish it.

## Review gate

Before Phase 7 publication work, the owner must verify the display name, role line,
contact method, GitHub URL, optional LinkedIn URL, resume facts, dates, and evidence
links. Missing values are a hard stop; placeholders and invented values are not
acceptable substitutes. The review result is recorded in this file's revision history
and reflected in implementation tests.
